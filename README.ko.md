# Git Change Trail

[English](README.md)

## 소개

Git Change Trail은 현재 편집 중인 파일의 커밋 이력을 VS Code 사이드바에서 탐색할 수 있는 확장 프로그램입니다. 커밋을 클릭하면 변경된 파일 목록이 디렉토리 트리로 표시되고, 특정 리비전의 파일 내용을 열거나 두 커밋을 나란히 비교할 수 있습니다.

## 기능

- **자동 추적** — 에디터 탭을 전환하면 Commit History 패널이 자동으로 갱신됩니다.
- **커밋 이력 목록** — 커밋이 최신 순으로 번호(`#1`, `#2`, …)와 함께 나열됩니다.
- **변경 파일 트리** — 커밋을 클릭하면 해당 커밋에서 변경된 파일이 디렉토리 트리로 표시됩니다.
- **리비전 파일 보기** — 트리의 파일을 클릭하면 해당 커밋 시점의 파일 내용을 읽기 전용으로 열 수 있습니다.
- **커밋 간 비교(diff)** — 파일 항목의 인라인 버튼으로 이전/이후 커밋과 파일을 비교합니다.
- **전체 파일 이력 보기** — 변경 파일 트리에서 파일 항목 버튼으로 해당 파일의 전체 이력을 바로 조회합니다.
- **날짜 필터** — 프리셋 또는 직접 입력으로 특정 날짜 범위의 커밋만 표시합니다.
- **저자 필터** — 선택한 저자의 커밋만 표시합니다.
- **핀 고정** — 패널을 현재 파일에 고정해 탭 전환에도 이력이 바뀌지 않도록 합니다.

## 요구 사항

- VS Code 1.90.0 이상
- Git CLI (`PATH`에 등록되어 있어야 합니다)

Git 저장소가 열린 VS Code 워크스페이스에서 동작합니다.

## 사용 방법

### 커밋 이력 보기

Git으로 추적 중인 파일을 에디터에서 열면 사이드바의 **Commit History** 패널이 자동으로 해당 파일의 이력을 표시합니다.

아직 열지 않은 파일의 이력을 보려면 Explorer에서 파일을 우클릭하고 **커밋 이력 보기**를 선택하세요.

### 커밋 탐색

Commit History 패널에서 커밋 항목을 클릭하면 **Commit Files** 패널에 해당 커밋에서 변경된 파일이 디렉토리 트리로 표시됩니다.

트리에서 파일을 클릭하면 해당 커밋 시점의 파일 내용이 읽기 전용으로 열립니다. 에디터 탭 제목에 짧은 커밋 해시가 포함되어 현재 파일과 구분됩니다.

### 커밋 간 비교(diff)

Commit Files 패널의 각 파일 항목에는 인라인 액션 버튼이 제공됩니다.

| 버튼 | 동작 |
|------|------|
| 이전 커밋과 비교 | 바로 이전 커밋과의 diff를 diff 에디터로 엽니다 |
| 이후 커밋과 비교 | 바로 이후 커밋과의 diff를 diff 에디터로 엽니다 |

diff가 열린 동안 Commit Files 패널 타이틀에 비교 중인 커밋 번호가 표시됩니다 (예: `comparing: #2 ↔ #3`).

### 파일 전체 이력 보기

Commit Files 패널의 파일 항목에서 **전체 파일 이력 보기** 버튼을 클릭하면 Commit History 패널이 해당 파일의 전체 커밋 이력으로 갱신됩니다. 이때 활성화된 필터는 초기화됩니다.

### 필터

Commit History 패널 타이틀 오른쪽에 세 개의 아이콘 버튼이 있습니다.

| 아이콘 | 동작 |
|--------|------|
| 달력 | 날짜 필터를 엽니다. `최근 7일`, `최근 30일` 프리셋을 선택하거나 `From`/`To` 날짜를 `YYYY-MM-DD` 형식으로 직접 입력합니다. 비워두면 해당 방향으로 제한을 두지 않습니다. **필터 초기화**를 선택해 필터를 제거할 수 있습니다. |
| 사람 | 저자 필터를 엽니다. 현재 파일 이력의 저자 목록에서 한 명 이상을 선택합니다. 모두 해제하면 필터가 초기화됩니다. |
| 핀 | 패널을 현재 파일에 고정합니다. 고정 중에는 탭을 전환해도 이력이 바뀌지 않습니다. 다시 클릭하면 자동 추적이 재개됩니다. |

필터가 적용된 상태에서는 패널 타이틀에 현재 날짜 범위 또는 선택 저자 수가 표시됩니다.

### 키보드 단축키

기본 단축키는 없습니다. VS Code **바로 가기 키** 설정에서 **커밋 이력 보기** 명령에 원하는 단축키를 지정할 수 있습니다.

## 개발 환경 설정

```bash
pnpm install
pnpm run compile
```

VS Code에서 이 저장소를 연 뒤 **Run Extension** 디버그 구성을 실행하면 Extension Development Host에서 확장을 확인할 수 있습니다.

자주 사용하는 명령어:

```bash
pnpm run compile      # TypeScript 단발 빌드
pnpm run watch        # 저장 시 자동 빌드
pnpm run lint         # ESLint 검사
pnpm run check        # lint + compile
pnpm run package:vsix # .vsix 설치 파일 빌드
```

## 기여하기

이슈, 버그 리포트, 기능 제안, 문서 개선 모두 환영합니다.

1. 저장소를 포크합니다.
2. 변경 내용을 별도 브랜치에서 작업합니다.
3. `pnpm run check`를 실행해 빌드가 통과하는지 확인합니다.
4. 변경 의도와 검증 방법을 포함해 Pull Request를 생성합니다.

버그를 제보할 때는 VS Code 버전, 운영체제, 재현 절차, 기대 동작과 실제 동작을 함께 적어주세요.
