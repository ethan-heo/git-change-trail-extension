# 요구사항 개발 계획서

## 목표

VSCode 사이드바에서 저자와 기간을 선택해 Git 변경 파일을 조회하고, 결과를 디렉토리 트리 형태로 탐색할 수 있게 한다.

## 주요 사용자 시나리오

1. 사용자는 Git 저장소가 열린 VSCode에서 Git Author Explorer 패널을 연다.
2. Author 입력란에 이름 또는 이메일 일부를 입력한다.
3. 매칭되는 저자 목록이 드롭다운에 표시된다.
4. 사용자는 한 명 이상의 저자를 선택한다.
5. From/To 날짜를 설정한다.
6. Search 버튼을 클릭한다.
7. 기간 내 선택 저자들이 수정한 파일이 통합 트리로 표시된다.
8. 파일을 클릭하면 에디터에서 일반 파일로 열린다.
9. 파일 행의 diff 버튼을 클릭하면 diff 뷰가 열린다.

## MVP 기능

| ID | 기능 | 구현 기준 |
|----|------|-----------|
| F-01 | 저자 검색 | 이름/이메일 부분 문자열로 필터링한다. |
| F-02 | 기간 설정 | From/To 날짜 입력을 지원한다. |
| F-03 | 저자 드롭다운 | 입력값과 매칭되는 저자 목록을 표시한다. |
| F-04 | 저자 복수 선택 | 선택된 저자를 태그 형태로 표시한다. |
| F-05 | 저자 선택 해제 | 태그의 삭제 버튼으로 개별 해제한다. |
| F-06 | 파일 트리 표시 | 선택 저자 전체 결과를 병합해 디렉토리 트리로 렌더링한다. |
| F-07 | 파일 열기 | 파일 클릭 시 `showTextDocument`로 연다. |
| F-08 | 빈 결과 처리 | 저자 미선택/검색 결과 없음 메시지를 표시한다. |
| F-09 | 오류 처리 | Git 미설치, 저장소 없음, 명령 실패를 안내한다. |
| F-10 | diff 열기 | 파일 행 호버 시 diff 버튼을 제공한다. |
| F-11 | 기간 기본값 | 최초 진입 시 From은 오늘 -7일, To는 오늘로 설정한다. |

## Post-MVP 기능

| ID | 기능 | 설명 |
|----|------|------|
| F-12 | 커밋 수 뱃지 | 파일별 해당 기간 내 커밋 횟수를 표시한다. |
| F-13 | 저자별 색상 구분 | 복수 저자 선택 시 파일 트리에서 저자별 시각 구분을 제공한다. |
| F-14 | 최근 검색 저장 | 마지막 검색 조건을 VSCode `globalState`에 저장한다. |

## 예외 처리

| 상황 | 사용자 메시지 |
|------|---------------|
| 저자를 선택하지 않고 검색 | 저자를 한 명 이상 선택해주세요. |
| 검색 결과 없음 | 해당 조건에 맞는 변경 파일이 없습니다. |
| Git 저장소 없음 | Git 저장소를 찾을 수 없습니다. |
| Git 명령 실패 | 오류 내용과 재시도 버튼을 표시한다. |
