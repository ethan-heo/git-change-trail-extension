## 2026-05-18 기능 전면 피벗

| 날짜 | 항목 | 결정 | 이유 |
|------|------|------|------|
| 2026-05-18 | 기능 방향 전환 | 저자 기반 검색 → 파일 기반 커밋 이력 탐색 | 코드 리뷰·분석 흐름에서 "이 파일에 무슨 일이 있었는가"가 더 직접적인 진입점이다. 기존 저자 검색 MVP는 폐기한다. |
| 2026-05-18 | 파일 선택 진입점 | 에디터 활성 파일 자동 추적 + 우클릭 컨텍스트 메뉴 조합 | 평상시 탭 전환만으로 이력이 갱신되어 손이 가장 덜 간다. 고정이 필요할 때는 핀 버튼, 명시적 트리거가 필요할 때는 우클릭 메뉴로 보완한다. |
| 2026-05-18 | 커밋 클릭 시 표시 범위 | 해당 커밋에서 변경된 파일만 표시 | 전체 저장소 트리보다 변경 범위에 집중하는 편이 코드 리뷰 맥락에 적합하다. git diff-tree 사용. |
| 2026-05-18 | 날짜 필터 UI 방식 | 패널 타이틀 아이콘 클릭 → QuickPick/InputBox | Tree View는 인풋 박스를 자체적으로 품지 못한다. VSCode 네이티브 QuickPick이 추가 구현 없이 가장 자연스럽다. |
| 2026-05-18 | 파일 전체 이력 진입점 | Commit Files 파일 항목 인라인 액션 | 특정 커밋에서 변경 파일을 보다가 같은 파일의 전체 변경 흐름으로 바로 전환할 수 있어 탐색이 끊기지 않는다. |
| 2026-05-18 | Diff 버튼 위치 | Commit Files 파일 항목 인라인 액션 | 사용자가 변경 파일 목록에서 바로 이전/이후 diff를 열 수 있고, 별도 중복 UI 없이 비교 동작을 제공한다. |
| 2026-05-18 | Webview 제거 | 검색 Webview 폐기, 모든 UI를 VSCode 네이티브 API로 대체 | 저자 검색 폼이 없어지므로 Webview가 불필요하다. Tree View + QuickPick + InputBox로 충분하다. |
