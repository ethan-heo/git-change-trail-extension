# QA 및 배포 개발 계획서

## QA 목표

Git Author Explorer의 핵심 검색 플로우가 실제 Git 저장소에서 안정적으로 동작하는지 확인한다.

## 수동 QA 체크리스트

| 항목 | 기대 결과 |
|------|-----------|
| Git 저장소에서 패널 열기 | Author 입력과 날짜 기본값이 표시된다. |
| Author 검색 | 이름/이메일 부분 검색으로 후보가 표시된다. |
| 저자 복수 선택 | 태그가 여러 개 표시되고 중복 선택되지 않는다. |
| 태그 삭제 | 선택된 저자 한 명만 제거된다. |
| 최근 7일 기본 검색 | From/To 기본값으로 검색된다. |
| 결과 있음 | 파일 트리가 디렉토리 구조로 표시된다. |
| 파일 클릭 | 에디터에서 파일이 열린다. |
| Commit Files 파일 항목의 전체 이력 버튼 클릭 | Commit History 패널이 해당 파일의 전체 커밋 이력으로 갱신된다. |
| Commit Files 파일 항목의 diff 버튼 클릭 | diff 뷰가 열린다. |
| 결과 없음 | 빈 결과 메시지가 표시된다. |
| 저자 0명 검색 | 저자 선택 안내 메시지가 표시된다. |
| Git 저장소 아님 | 저장소 없음 메시지가 표시된다. |

## 자동 테스트 범위

- Git stdout 파싱
- 저자 검색 필터링
- 파일 목록 병합 및 중복 제거
- 경로 배열의 트리 변환
- 날짜 기본값 계산
- Webview 메시지 payload 검증

## 테스트 저장소 케이스

| 케이스 | 목적 |
|--------|------|
| 작은 저장소 | 기본 동작 빠른 확인 |
| 커밋이 많은 저장소 | GitService 성능 확인 |
| 모노레포 | 깊은 디렉토리 트리 렌더링 확인 |
| 저자 이메일이 여러 개인 저장소 | 저자 표시와 검색 정확도 확인 |
| 변경 파일이 삭제된 저장소 | 존재하지 않는 파일 클릭 처리 확인 |

## 패키징

```bash
vsce package
```

패키징 전 확인:

- README에 기능 설명과 사용법이 있다.
- Marketplace 표시용 icon과 categories가 설정되어 있다.
- `engines.vscode` 최소 버전이 실제 사용 API와 맞는다.
- `.vscodeignore`가 빌드 산출물 기준으로 정리되어 있다.

## 배포 후 확인

- `.vsix` 설치 후 사이드바 view가 나타나는지 확인한다.
- 새 창에서 Git 저장소를 열고 검색 플로우를 다시 검증한다.
- Marketplace 게시 시 버전 태그와 CHANGELOG를 함께 남긴다.
